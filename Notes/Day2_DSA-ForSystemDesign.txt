1. Linked List

A linked list is a linear data structure where elements (nodes) are connected using pointers. Each node contains:

* Data (value)
* Reference to the next node

Unlike arrays, memory is not continuous.

2. Types of Linked List

a. Singly Linked List

Each node points to the next node.


1 → 2 → 3 → 4 → NULL


* Traversal is only forward
* Simple structure

b. Doubly Linked List

Each node has two pointers:

* Previous node
* Next node


NULL ← 1 ⇄ 2 ⇄ 3 ⇄ 4 → NULL


* Allows backward traversal
* Uses more memory

c. Circular Linked List

Last node connects back to the first node.


1 → 2 → 3 → 4
↑           ↓
← ← ← ← ← ←


* No NULL pointer
* Useful in cyclic processes

d. Circular Doubly Linked List

Combination of circular and doubly linked list.

---

3. Important Concepts in Linked List

a. Slow and Fast Pointer Technique

Used to:

* Find middle element
* Detect cycles

Logic:

* Slow pointer moves 1 step
* Fast pointer moves 2 steps

When fast reaches end, slow is at middle.

b. Cycle Detection (Missing but Important)

If slow and fast pointers meet, a cycle exists.

---

4. Reverse Linked List

Goal: reverse direction of pointers

Steps:

1. Keep three pointers: prev, current, next
2. Reverse links one by one

Time Complexity: O(n)

---

5. Palindrome Linked List

Check if linked list reads same forward and backward.

Steps:

1. Find middle
2. Reverse second half
3. Compare both halves

Important addition:

* Restore the list after checking (good practice)

1. Stack

Stack follows LIFO (Last In First Out).

Operations:

* Push (insert)
* Pop (remove top element)
* Peek (view top element)

Use cases:

* Function calls (call stack)
* Undo/redo operations
* Expression evaluation

---

2. Parenthesis Matching Problem

Use stack to validate expressions.

Logic:

* Push opening brackets
* For closing bracket, check top of stack

Important cases:

* Different types: (), {}, []
* If stack is empty and closing appears → invalid

---

1. Queue

Queue follows FIFO (First In First Out).

Operations:

* Enqueue (insert)
* Dequeue (remove)


Front → [1, 2, 3, 4] → Rear


---

2. Types of Queue

a. Simple Queue

Basic FIFO

b. Deque (Double Ended Queue)

Insertion and deletion from both ends

c. Priority Queue

Elements are processed based on priority, not order

---

d. Missing Concept: Circular Queue

* Avoids memory wastage in array implementation
* Front and rear wrap around

# 1. Cycle Detection in Linked List

A cycle exists when a node points back to a previous node instead of NULL.

=> Example


1 → 2 → 3 → 4 → 5
          ↑     ↓
          ← ← ←


Here, node 5 points back to node 3.

---

=> How to Detect Cycle (Floyd’s Algorithm)

Use two pointers:

* Slow → moves 1 step
* Fast → moves 2 steps

 Logic:

* If there is no cycle → fast reaches NULL
* If there is a cycle → slow and fast will meet


Step 1:
Slow = 1, Fast = 1

Step 2:
Slow = 2, Fast = 3

Step 3:
Slow = 3, Fast = 5

Step 4:
Slow = 4, Fast = 4  → they meet → cycle detected


---

=> Finding Start of Cycle (Advanced)

After meeting:

1. Move one pointer to head
2. Move both one step at a time
3. Where they meet again = start of cycle

---

=> Why Important

* Detect infinite loops
* Used in memory structures and system debugging

---

# 2. Circular Queue

A circular queue solves the limitation of a normal queue (wasted space).

=> Problem in Normal Queue


[ ][ ][ ][ ][ ]
 F       R

After dequeues:
[ ][ ][ ][ ][ ]
     F     R

Front moved forward but space is wasted


---

=> Circular Queue Idea

Rear wraps around to the beginning.


[1][2][3][4][5]
 F           R

After operations:

[6][2][3][4][5]
 R   F


---

=> Key Conditions

* Queue is FULL when:


(rear + 1) % size == front


* Queue is EMPTY when:


front == -1


---

=> Why Important

* Used in:

  * CPU scheduling
  * Buffers (stream processing)
  * Real-time systems

# 3. HashMap Internals

You know HashMap stores key-value pairs. Now understand how it works internally.

---

=> Step 1: Hash Function

Converts key into index


index = hash(key) % size


Example:


key = 12
index = 12 % 5 = 2


---

=> Step 2: Collision

Two keys can map to same index.

Example:


12 % 5 = 2
17 % 5 = 2


---

=> Collision Handling

 1. Chaining

Store multiple elements in same bucket (linked list)


Index 2 → [12 → 17 → 22]


---

 2. Open Addressing (basic idea)

Find another empty slot

---

=> Load Factor


load factor = number of elements / size of array


When load increases:

* HashMap resizes (rehashing)

---

=> Time Complexity

* Average: O(1)
* Worst: O(n)

---

=> Why Important

Used in:

* Caching (Redis)
* Databases
* Fast lookups

# 4. AVL Tree Rotations

AVL Tree is a self-balancing BST.

Balance condition:


|height(left) - height(right)| ≤ 1


If violated → perform rotations

---

=> Case 1: Left-Left (LL Rotation)

Problem:


    30
   /
  20
 /
10


Solution: Right Rotation


    20
   /  \
 10   30


---

=> Case 2: Right-Right (RR Rotation)

Problem:


10
  \
   20
     \
      30


Solution: Left Rotation


    20
   /  \
 10   30


---

=> Case 3: Left-Right (LR Rotation)

Problem:


    30
   /
  10
    \
     20


Steps:

1. Left rotate (10)
2. Right rotate (30)

Result:


    20
   /  \
 10   30


---

=> Case 4: Right-Left (RL Rotation)

Problem:


10
  \
   30
   /
  20


Steps:

1. Right rotate (30)
2. Left rotate (10)

Result:


    20
   /  \
 10   30


---

=> Why AVL Matters

* Keeps tree balanced
* Ensures O(log n) operations
